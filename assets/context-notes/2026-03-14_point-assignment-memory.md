## What

이 문서는 `point-api` 프로젝트에서 계속 기준으로 삼아야 하는 과제 목적, 필수 기능, 예시 시나리오, 권장 스키마, 구현 원칙을 정리한 메모다.
이후 구현 시 이 문서를 기준으로 적립, 적립취소, 사용, 사용취소 기능을 확장한다.

## Why

요구사항이 길고 세부 제약이 많아서 구현 중 일부 조건이 빠질 가능성이 있다.
핵심 요구사항과 예시 흐름을 고정된 문서로 남겨 두면 이후 기능 확장 시 일관성을 유지할 수 있다.

## Required Features

### 1. 적립

1. 1회 적립가능 포인트는 1포인트 이상, 10만포인트 이하로 가능하며 1회 최대 적립가능 포인트는 하드코딩이 아닌 방법으로 제어할 수 있어야 한다.
2. 개인별로 보유 가능한 무료포인트의 최대금액 제한이 존재하며, 하드코딩이 아닌 별도의 방법으로 변경할 수 있어야 한다.
3. 특정 시점에 적립된 포인트는 1원 단위까지 어떤 주문에서 사용되었는지 추적할 수 있어야 한다.
4. 포인트 적립은 관리자가 수기로 지급할 수 있으며, 수기지급한 포인트는 다른 적립과 구분되어 식별할 수 있어야 한다.
5. 모든 포인트는 만료일이 존재하며, 최소 1일 이상 최대 5년 미만의 만료일을 부여할 수 있다. 기본값은 365일이다.

### 2. 적립 취소

1. 특정 적립행위에서 적립한 금액만큼 취소 가능해야 한다.
2. 적립한 금액 중 일부가 사용된 경우라면 적립 취소될 수 없다.

### 3. 사용

1. 주문 시에만 포인트를 사용할 수 있다고 가정한다.
2. 포인트 사용 시에는 주문번호를 함께 기록하여 어떤 주문에서 얼마의 포인트를 사용했는지 식별할 수 있어야 한다.
3. 포인트 사용 시에는 관리자가 수기 지급한 포인트가 우선 사용되어야 하며, 만료일이 짧게 남은 순서로 사용해야 한다.

### 4. 사용 취소

1. 사용한 금액 중 전체 또는 일부를 사용취소할 수 있다.
2. 사용취소 시점에 이미 만료된 포인트를 사용취소해야 한다면 그 금액만큼 신규 적립 처리해야 한다.

## Example Scenario

1. 1000원 적립한다. 총 잔액은 `0 -> 1000`원이다.
   - `pointKey`: `A`
2. 500원 적립한다. 총 잔액은 `1000 -> 1500`원이다.
   - `pointKey`: `B`
3. 주문번호 `A1234` 에서 1200원 사용한다. 총 잔액은 `1500 -> 300`원이다.
   - `pointKey`: `C`
   - A 적립에서 1000원 사용한다. A의 사용 가능 잔액은 `1000 -> 0`
   - B 적립에서 200원 사용한다. B의 사용 가능 잔액은 `500 -> 300`
4. A의 적립이 만료되었다.
5. C의 사용금액 1200원 중 1100원을 부분 사용취소 한다. 총 잔액은 `300 -> 1400`원이다.
   - `pointKey`: `D`
   - 1200원은 A와 B에서 사용되었다.
   - 사용취소 시 A의 사용 가능 잔액은 원래 `0 -> 1000`원이 되어야 하지만, A는 이미 만료되었으므로 `pointKey E` 로 1000원이 신규 적립되어야 한다.
   - B는 만료되지 않았기 때문에 사용 가능 잔액은 `300 -> 400`원이 된다.
   - C는 이제 1200원 사용금액 중 100원을 부분취소 할 수 있다.

## Recommended Schema

아래 스키마는 현재 과제 요구사항을 만족하기 위한 권장 기준이다.

### product

```sql
CREATE TABLE product (
    product_id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    name VARCHAR(255) NOT NULL,
    price_won BIGINT NOT NULL CHECK (price_won >= 0),
    stock_qty INT NOT NULL CHECK (stock_qty >= 0),
    version BIGINT NOT NULL DEFAULT 0
);
```

### orders

```sql
CREATE TABLE orders (
    order_id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    order_no VARCHAR(50) NOT NULL UNIQUE,
    user_id BIGINT NOT NULL,
    request_key VARCHAR(64) NOT NULL UNIQUE,
    total_amount_won BIGINT NOT NULL CHECK (total_amount_won >= 0),
    point_usage_amount_won BIGINT NOT NULL DEFAULT 0 CHECK (point_usage_amount_won >= 0),
    payment_amount_won BIGINT NOT NULL CHECK (payment_amount_won >= 0),
    ordered_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    CHECK (payment_amount_won = total_amount_won - point_usage_amount_won)
);
```

### point_policy

```sql
CREATE TABLE point_policy (
    policy_code VARCHAR(30) PRIMARY KEY,
    max_earn_per_tx_won BIGINT NOT NULL CHECK (max_earn_per_tx_won BETWEEN 1 AND 100000),
    max_balance_won BIGINT NOT NULL CHECK (max_balance_won >= 0),
    default_expire_days INT NOT NULL CHECK (default_expire_days >= 1),
    min_expire_days INT NOT NULL CHECK (min_expire_days >= 1),
    max_expire_days INT NOT NULL CHECK (max_expire_days < 1825),
    CHECK (min_expire_days <= default_expire_days),
    CHECK (default_expire_days <= max_expire_days)
);
```

### point_wallet

```sql
CREATE TABLE point_wallet (
    user_id BIGINT PRIMARY KEY,
    balance_won BIGINT NOT NULL DEFAULT 0 CHECK (balance_won >= 0),
    version BIGINT NOT NULL DEFAULT 0
);
```

### point_ledger

```sql
CREATE TABLE point_ledger (
    point_id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    point_key VARCHAR(64) NOT NULL UNIQUE,
    request_hash VARCHAR(64) NOT NULL,
    user_id BIGINT NOT NULL,
    tx_type VARCHAR(20) NOT NULL CHECK (tx_type IN (
        'EARN',
        'ADMIN_GIFT',
        'EARN_CANCEL',
        'USE',
        'USE_CANCEL',
        'CANCEL_RETURN',
        'EXPIRE'
    )),
    amount_won BIGINT NOT NULL CHECK (amount_won > 0),
    remained_amount_won BIGINT NOT NULL DEFAULT 0 CHECK (remained_amount_won >= 0),
    expired_at TIMESTAMP,
    order_id BIGINT,
    related_point_id BIGINT,
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (order_id) REFERENCES orders(order_id),
    FOREIGN KEY (related_point_id) REFERENCES point_ledger(point_id),
    CHECK (remained_amount_won <= amount_won),
    CHECK (
        (tx_type IN ('EARN', 'ADMIN_GIFT', 'CANCEL_RETURN') AND expired_at IS NOT NULL)
        OR
        (tx_type IN ('EARN_CANCEL', 'USE', 'USE_CANCEL', 'EXPIRE'))
    ),
    CHECK (
        (tx_type IN ('EARN', 'ADMIN_GIFT', 'CANCEL_RETURN') AND remained_amount_won BETWEEN 0 AND amount_won)
        OR
        (tx_type IN ('EARN_CANCEL', 'USE', 'USE_CANCEL', 'EXPIRE') AND remained_amount_won = 0)
    ),
    CHECK (
        (tx_type = 'USE' AND order_id IS NOT NULL)
        OR
        (tx_type <> 'USE')
    )
);
```

### point_usage_detail

```sql
CREATE TABLE point_usage_detail (
    usage_detail_id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    point_id BIGINT NOT NULL,
    detail_type VARCHAR(30) NOT NULL CHECK (detail_type IN (
        'USE',
        'USE_CANCEL_RESTORE',
        'USE_CANCEL_REGRANT'
    )),
    order_id BIGINT NOT NULL,
    source_point_id BIGINT NOT NULL,
    target_point_id BIGINT,
    amount_won BIGINT NOT NULL CHECK (amount_won > 0),
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (point_id) REFERENCES point_ledger(point_id),
    FOREIGN KEY (order_id) REFERENCES orders(order_id),
    FOREIGN KEY (source_point_id) REFERENCES point_ledger(point_id),
    FOREIGN KEY (target_point_id) REFERENCES point_ledger(point_id),
    CHECK (
        (detail_type IN ('USE', 'USE_CANCEL_RESTORE') AND target_point_id IS NULL)
        OR
        (detail_type = 'USE_CANCEL_REGRANT' AND target_point_id IS NOT NULL)
    ),
    UNIQUE (point_id, detail_type, source_point_id)
);
```

### Indexes

```sql
CREATE INDEX idx_orders_user_id_ordered_at
    ON orders (user_id, ordered_at);

CREATE INDEX idx_point_ledger_user_created_at
    ON point_ledger (user_id, created_at);

CREATE INDEX idx_point_ledger_user_type_expired
    ON point_ledger (user_id, tx_type, expired_at, point_id);

CREATE INDEX idx_point_ledger_related_point_id
    ON point_ledger (related_point_id);

CREATE INDEX idx_point_usage_detail_order_id
    ON point_usage_detail (order_id);

CREATE INDEX idx_point_usage_detail_source_point_id
    ON point_usage_detail (source_point_id);

CREATE INDEX idx_point_usage_detail_point_id
    ON point_usage_detail (point_id);
```

## Implementation Guidance

- 한국어가 깨지지 않도록 주의한다.
- 정책값은 하드코딩하지 않고 테이블 또는 설정으로 관리한다.
- 문자열 상수 남발보다 `enum` 사용을 우선한다.
- 포인트 거래 유형과 사용 상세 유형은 `enum` 으로 관리한다.
- 적립 lot 추적과 사용 상세 추적이 핵심이므로 원장과 상세 테이블의 연결을 유지한다.
- 이후 구현은 적립, 적립취소, 사용, 사용취소 순으로 확장한다.
